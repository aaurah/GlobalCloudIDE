---
name: Orval schemas conflict
description: The `schemas` option in orval zod config generates TypeScript interfaces that clash with Zod schema value exports, causing TS2308. Also, the barrel index.ts is auto-regenerated by orval on each codegen run.
---

## Rule
Do NOT use `schemas: { path: "...", type: "typescript" }` in the orval `zod` output config. It generates TypeScript interfaces with the same names as the Zod schemas (e.g. `CreateProjectBody`), which causes TS2308 "already exported" errors when both are re-exported from the same barrel.

**Why:** Orval generates both a Zod schema (`export const Foo = zod.object(...)`) and a TypeScript interface (`export interface Foo {...}`) with the same name. When the barrel does `export * from "./api"` and `export * from "./types"`, TypeScript fires TS2308. Using `export type *` for the types does NOT fix it — the interface is a value export too.

**How to apply:** Remove the `schemas` option from `orval.config.ts` zod section. Types are derivable via `z.infer<typeof Schema>`. Also note that `lib/api-zod/src/index.ts` is OVERWRITTEN by orval on each codegen run — manual edits to it are lost. Configure orval to control what goes in the barrel, not the barrel file itself.
