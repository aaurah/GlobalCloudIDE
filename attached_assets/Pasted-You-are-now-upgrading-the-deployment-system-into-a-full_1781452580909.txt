### NEW FEATURES TO ADD

1. **Environment System**
   - Add support for:
       - development
       - staging
       - production
   - Each environment has:
       - its own config
       - its own deployments
       - its own URLs
   - Endpoints:
       - /envs/list
       - /envs/config
       - /deploy/{env}/start
       - /deploy/{env}/status

2. **Blue-Green Deployments**
   - Implement blue-green strategy:
       - Blue = current live version
       - Green = new version
   - Steps:
       1. Deploy Green alongside Blue
       2. Run health checks on Green
       3. Switch traffic from Blue → Green
       4. Keep Blue as rollback target
   - Add:
       - /deploy/blue-green/start
       - /deploy/blue-green/switch
       - /deploy/blue-green/rollback

3. **Canary Releases**
   - Implement canary rollout:
       - Gradual traffic shifting:
           - 1% → 10% → 25% → 50% → 100%
   - Monitor:
       - error rate
       - latency
       - resource usage
   - Auto-abort if metrics degrade.
   - Endpoints:
       - /deploy/canary/start
       - /deploy/canary/progress
       - /deploy/canary/abort

4. **Rollbacks**
   - Track deployment history:
       - version
       - commit hash
       - timestamp
       - environment
   - Allow:
       - one-click rollback
       - rollback with reason
   - Endpoints:
       - /deploy/history
       - /deploy/rollback

5. **Secrets & Environment Variables**
   - Add secure storage for:
       - API keys

       - tokens
   - Support:
       - per-environment secrets
       - per-project secrets
   - Endpoints:
       - /secrets/set
       - /secrets/get

   - Inject secrets into runtime at deploy time.

6. **Build & Release Pipelines**
